# 什么是闭包？它有哪些实际应用？

## 一句话定义

**闭包（Closure）= 函数 + 它声明时所在的词法作用域。**

即使外层函数已经执行结束，内部函数依然可以访问外层作用域中的变量。

---

## 面试里推荐怎么答

可以按这个顺序讲：

1. **先给定义**：函数在创建时会“记住”它所在的词法环境。
2. **再说本质**：不是“把变量复制进来”，而是“内部函数仍然引用那块作用域”。
3. **最后给场景**：数据私有化、函数工厂、缓存、防抖节流、事件处理器等。

> 面试官通常不只想听概念，更想看你能不能把它和真实业务联系起来。

---

## 经典示例

```js
function createCounter() {
  let count = 0;

  return function () {
    count += 1;
    return count;
  };
}

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

### 这里为什么是闭包？

- `createCounter` 执行完之后，按理说它的局部变量应该被销毁。
- 但返回出去的内部函数还在引用 `count`。
- 所以 `count` 对应的那部分作用域不会立刻释放。
- 这就形成了闭包。

## 真实业务里的常见应用

### 1. 数据私有化

```js
function createStore() {
  let state = { token: '' };

  return {
    setToken(token) {
      state.token = token;
    },
    getToken() {
      return state.token;
    },
  };
}
```

适合回答：

- “不想让外部直接改内部变量”
- “只暴露有限接口”

### 2. 函数工厂 / 配置复用

```js
function createValidator(minLength) {
  return function (value) {
    return value.length >= minLength;
  };
}

const validatePassword = createValidator(8);
```

这类场景很常见：

- 表单校验
- 接口封装
- 权限判断器
- 埋点函数预配置

### 3. 防抖 / 节流

```js
function debounce(fn, delay) {
  let timer = null;

  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

这里的 `timer` 能一直被记住，就是因为返回函数形成了闭包。

### 4. 缓存结果

```js
function memoize(fn) {
  const cache = new Map();

  return function (...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);

    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}
```

这类题很适合继续往“性能优化”方向展开。

## 容易被追问的点

### 闭包会不会导致内存泄漏？

**闭包本身不是内存泄漏。**

真正的问题是：

- 闭包长期持有不再需要的大对象
- 事件监听、定时器、DOM 引用没有及时释放
- 导致本该被回收的对象继续被引用

所以更准确的说法应该是：

> 闭包可能让对象生命周期变长，如果使用不当，会放大内存泄漏风险。

### `for` 循环里为什么经常出问题？

经典原因是：多个函数可能共享了同一个外层变量。

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 3 3 3
```

因为 `var` 没有块级作用域，三个回调引用的是同一个 `i`。

更好的写法：

```js
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
```

## 一段更完整的面试回答模板

你可以直接按这个思路说：

> 闭包本质上是函数和它声明时词法作用域的组合。即使外层函数执行完了，内部函数仍然可以访问外层变量。实际开发里我比较常见的使用场景有数据私有化、函数工厂、防抖节流、缓存等。它的好处是能封装状态，但如果闭包长期持有大对象、DOM 或定时器引用，也可能导致内存占用过高，所以要注意及时释放无用引用。

---

## 结尾加分点

如果面试官还在追问，你可以继续补一句：

- **从 JS 引擎角度看**，闭包和词法环境、作用域链有关。
- **从工程角度看**，闭包最重要的价值是“保存状态”和“延长变量生命周期”。
